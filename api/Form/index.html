Form

A state object that helps when creating UIs that depend too much on user input.

Types​

Submission​

</>
interface Submission {
id: string
valid: boolean
post: boolean
data: {[string]:any}
error: {[string]:string}
}

Functions​

clear​

</>
Form:clear() → ()

Clears the last result and all property values.

field​

</>
Form:field(name: string) → PubTypes.Value<any>

Creates a value bound to a field inside the Form. Can be used alongside Out to get user input.

submit​

</>
Form:submit() → Submission

Submits the form, returning the result of the submission.

get​

</>
Form:get() → Submission

Returns the result of the last submission, submitting one if it was cleared or does not exist.

onSubmit​

</>
Form:onSubmit(fn: (result:PubTypes.Submission) → ()) → () → ()

Binds a callback for when the form is submitted, the callback will still be ran if there are validation errors on the result. Returns a function that disconnects the callback from the signal.

onPost​

</>
Form:onPost(fn: (data: {[string]:any}) → ()) → () → ()

Binds a callback for when the form is posted, meaning it is completely fine with zero fatal errors from the validators. Returns a function that disconnects the callback from the signal.

onError​

</>
Form:onError(fn: (data: {[string]:string}) → ()) → () → ()

Binds a callback that runs whenever an validation error (fatal or non-fatal) on a form field fails, passing the errors that the form submission had.
